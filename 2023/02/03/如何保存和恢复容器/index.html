如何保存和恢复容器？

容器在一台机器上跑着，如果想把它切到其它机器上继续跑，同时保持之前的状态，应该怎么做？我们来学习一下如何保存一个容器。以下是多种方法

docker commit + save

这种方式，commit容器，作为一个image，然后save成一个文件。

docker commit

把容器的文件修改保存下来，形成一个新的image。这种方式不会包含volumes中的数据。

以上只是把container保存成image，无法传输。

docker save

此种方式可以把image保存称为文件，可以传输。

然后load和查看容器内的文件。这样就相当于把一个container转移到另一个服务器

可以看出此方式也可以把容器转移到另一个服务器

对比

以上方式看起来很想。那么它们的区别是什么？
查阅文档，可知docker export是Export a container’s filesystem as a tar archive，而docker save是Save one or more images to a tar archive (streamed to STDOUT by default)。前者只是保存文件系统，而后者保存的是image。

前者缺少了一些信息，例如env, cmd, image。为了保存原始信息，还是使用docker commit比较好。
